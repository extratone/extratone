In practice, adding it in a quick and speedy way means there will be software out there that will be written to not be able to handle more than one, much like the current state of the type property. Then there‚Äôs also the practical question of attempting to disambiguate which value in the signatureAlgorithm list corresponds to which use case.

IMO, what is missing is a one-to-many mapping from a publicKey to key-use-cases, and then a one-to-one mapping from each key-use-case to metadata (signatureAlgorithm in the HTTP Signatures use-case). And it‚Äôs a matter of finding an expression that makes it clear to implementing devs that these are the N-ary expectations.
