nightpool July 14, 2020, 1:10pm #7

Mastodon does not return 202 if the signature is valid, although we would be well within our rights to per the spec. But for HTTP signatures specifically we always process those synchronously and return a 401 with a specific error reason. Like the spec says, we return 202 if we accept the payload for further asynchronous processing, regardless of whether that processing succeeds or not. (since it‚Äôs asynchronous, there‚Äôs no way to know)

(If you read the actual issue you linked, you‚Äôd know that this is the case, since it concludes by saying "We know now that 202 is okay in terms of signature)
