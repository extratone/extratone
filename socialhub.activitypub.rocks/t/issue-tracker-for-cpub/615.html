I just enabled the issue tracker on GitLab: https://gitlab.com/openengiadina/cpub/-/issues

Seems to be a issue with the mix ecto.create task with peer authentication. Unfortunately, I think the solution is to change the PostgreSQL config to allow password authentication from localhost. See also this issue on the Pleroma issue tracker: https://git.pleroma.social/pleroma/pleroma/issues/232.
